# AdPilot V3: Mojo Performance Agent

## Overview
AdPilot V3 (Mojo) is a next-generation **multi-tenant ads intelligence and management platform**. It is designed to act as a "Performance Agent" that bridges the gap between raw ad data and strategic action. By unifying Meta and Google Ads data into a single, high-performance dashboard, AdPilot enables growth teams to scale budgets efficiently while maintaining 95%+ data accuracy.

---

## Why AdPilot?
Most ad managers spend 80% of their time on reporting and only 20% on strategy. AdPilot reverses this:
- **Automated Intelligence**: Python agents pull live API data hourly, identifying winners and losers based on statistical thresholds.
- **Full-Funnel Tracking**: Beyond just clicks, AdPilot tracks "Positive Leads," "Site Visits (SVs)," and "Closures" to calculate true ROI (CPQL, CPSV).
- **Audit-First Culture**: Every change made (pausing a campaign, scaling a budget) requires a **Strategic Rationale**, building a historical learning loop for the team.

---

## Core Features

### 1. Multi-Client Registry
Manage an unlimited number of clients from a single interface.
- **Independent Credentials**: Securely store Meta Access Tokens and Google OAuth keys per client.
- **Platform Toggles**: Enable or disable Meta/Google integrations on a per-client basis.
- **Persistent Data**: Data for each client is stored in an independent filesystem registry (`ads_agent/data/`).

### 2. The Command Center (Dashboard)
A premium, dark-mode interface providing the "pulse" of the account.
- **Health Scores (0-100)**: Proprietary scoring that weighs CPL, CTR, CVR, and Pacing.
- **Funnel Split**: Radial charts showing conversion distribution across the funnel (TOFU/MOFU/BOFU).
- **Tracking Sanity**: Automated alerts if "Zero Leads" are detected for active spend, preventing tracking leaks.

### 3. Unified Actions & Strategic Rationale
Execute optimizations directly from the dashboard.
- **Auto-Execute**: Pause underperformers or scale winners via API integration.
- **Rational Logging**: Every action requires a "Strategic Call" (rationale), which is logged to `learning-data.csv` for future AI training.
- **Cooldown SOP**: Avoids over-optimization by showing "Last Action" time-ago to prevent constant tweaking.

### 4. Specialized Panels
- **Creative Calendar**: Visibility into creative fatigue and rotation schedules.
- **Google Bidding & QS**: Deep-dive into Google Ads impression share, lost-rank/lost-budget, and Quality Scores.
- **MTD Deliverables**: A section for entering offline metrics (SVs, Closures) to calculate "Cost per Quality Lead."

## How to Use AdPilot

### Step 1: Client Onboarding
1. Navigate to **Manage Clients** in the sidebar.
2. Click **Add Client** and enter the client name and target locations.
3. Click the **Credentials** icon on the new client row.
4. Enter the required API keys (Meta Token, Google MCC ID, etc.).

### Step 2: Data Population
The dashboard requires analysis data generated by the Python agents.
1. Run the Meta Agent: `python3 ads_agent/meta_ads_agent_v2.py`
2. Run the Google Agent: `python3 ads_agent/google_ads_agent_v2.py`
*Note: These are typically automated via a daily cron schedule at 9:00 AM IST.*

### Step 3: Analysis & Optimization
1. Select the client from the **Client Selector** in the sidebar.
2. Review the **Health Score** and **Critical Alerts** at the top.
3. Use the **Campaign Health** table to identify "LOSERS" (CPL > Target).
4. Click **Pause** or **Scale** on a campaign — you will be prompted to enter your **Strategic Rationale**.

### Step 4: Tracking Offline Metrics
Towards the end of the month, navigate to **MTD Deliverables**.
1. Enter the number of **SVs (Site Visits)** and **Closures** achieved.
2. Review the derived **CPQL** and **CPSV** metrics to assess month-over-month efficiency.

## Technical Stack
- **Frontend**: Vite + React + Tailwind CSS + Lucide Icons
- **Backend**: Node.js + Express (handling registry, credentials, and API proxying)
- **Data Engine**: Python 3.x (Facebook Business SDK & Google Ads API)
- **Persistence**: JSON-backed local storage for registries; Encrypted credential store.
