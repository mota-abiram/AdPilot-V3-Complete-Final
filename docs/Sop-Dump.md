2) Decide CPL (the only “thinking” input)


Start with ticket-size band, then adjust for brand and location. Use the lower end if brand is strong and/or location is popular.


Baseline CPL bands (₹)


₹40–60L: 550–750


₹60–80L: 650–850


₹80L–1.5Cr: 750–950


₹1.5–2Cr: 850–1,000


₹2–4Cr: 1,000–1,400

Adjustment rules


Brand = High: shift –10% (toward lower limit)


Brand = Low: shift +10% (toward upper limit)


Location = High: shift –10%; Low: +10%


Cap total adjustment at ±15% from the band midpoint.

Do NOT use one Google Ads conversion tag with three different triggers (TP + phone + WhatsApp).
That approach can inflate counts, complicate attribution, and makes optimization muddy.

Best practice:
Keep Lead (Thank-You Page) as a Primary conversion.


Create separate Google Ads conversion actions (Secondary) for:



Phone Call – Click (tel: link)


Contact – WhatsApp Click


Create separate GTM tags for each conversion action, each with its own trigger.

2) Creative principles (what to show)
Price + Place + Proof on-frame
Put ticket price (or band), location, and 1–3 quantified USPs, detailed config right on the image/video (clubhouse area, towers/floors/units, commute mins). People don’t want fluff—specifics win.
Highlight the core offer
 Clearly highlight the product/service/offering that you're advertising. Though this looks simple, but we tend to miss out on highlighting our core offer properly. One of the most common pitfalls to avoid.
Urgency & FOMO
Real deadlines / inventory scarcity drive action; refresh offers periodically (don’t run the same sale forever). Have it clearly displayed.
Show benefits & the problem
Paint the outcome (benefits) or the pain if they delay (problem-first frames). Make the problem sound big, make people feel shitty about their problem. You're hero as big as the villain. Ex - For fitness brand, you can highlight that you feel lethargic & tired if you don't go to the gym
Relatable beats over-produced
“Amateur”/raw look often outperforms high-gloss; keep it real when brand permits. Instagram fonts, raw imagery, basically the goal is NOT to look Ad like.

3) Static design rules (do these on every image)
Avoid Facebook-like blues/whites; use contrasting colors (bright yellow, neon green, crimson, orange, magenta) to pop in feed/dark mode. 
Use green ticks to list benefits (tested to lift CTR), especially on women-skewed sets. 
Do NOT add arrows, mouse cursors, fake play buttons in the creative—risk disapprovals and higher CPC. 
Faces? Zoom to the eyes—humans process eyes first; improves read & click-through when the rest matches intent. 
Consider collages (simple Canva grids) to pattern-interrupt. 
Consider sequential carousels that glide frame-to-frame (AIDA). 
Consider Instant Experience for an on-platform mini-LP (often cheaper; reduces overall CPA). 


4) Policy & sanity (Meta safe list)
Allowed & effective: green ticks; bold offers; on-frame price; testimonials (avoid unsubstantiated claims). 
Risky / avoid: arrows, mouse cursors, fake UI, fake play icons; claim overreach. 
Keep sizes native (don’t rely on auto-resize). 


5) Video creation rules (35–50s)
Make it feel like a Reel. Lead with movement (coming in the frame, zooming on the face, sitting, walking etc) in ≤3s; keep it “native” and vertical.
First 3 Words of the video should address the your target audience; if going for a detailed subset, it should address your detailed buyer persona. For Ex - Hey IT Folks - Target Audience. Stop if you're a software engineer - Buyer persona.
Hook in first 3 seconds (movement/text/contrast).
Pattern-interrupts: VCR/reverse, jump cuts, B-roll, mid-roll switch; strong top headline, captions, progress bar (works on mute).
Story arc (AIDA): attention → problem/benefit → visualize outcome → CTA + FOMO.
Mute-friendly: captions always—lots of users watch without sound.
(These tactics echo the guide’s best-performing ad patterns around movement, cuts, headline/captions/progress, and storytelling.)

8) Diagnostics & success metrics (read at ad level)
Order of diagnosis (what to fix first):
First-frame Rate → target ≥90%; if low: fix thumbnails, first 3 words, contrast. 
Thumb-stop (3s views ÷ impressions) → target ≥30%; if low: fix hook/thumbnail/contrast.
Hold 3→15s (aim ≥35% retention through the meat) → tighten the middle; add pattern-interrupt, core messaging to be clear, highlight proper benefits.
CTR (link) → if weak, sharpen CTA/offer/benefits.
CPM high with weak CTR → audience/creative fit or fatigue → refresh.
(This performance tree matches the guide’s approach: diagnose CPM and CTR changes and refresh creatives when frequency rises and CTR drops.)


9) Refresh & “Revive” playbook
If performance dips, wait ~3 to 5 days; if not back, refresh creatives (fatigue).
Fatigue signs: rising frequency (≈>2.5 TOFU/MOFU; >4 BOFU), falling CTR, rising CPM.
Try cloning winning creatives from other campaigns into this ad set before net-new production.
Minor frame tweaks or light animation can reset learning and open new pockets.


10) Production guardrails
Don’t stuff everything into one frame—pick a single idea per creative (price, benefit, social proof, or offer).
Multiple placements ⇒ multiple dimensions (produce all sizes up front). 
Keep one experiments campaign running to farm next-gen winners (so you have backups when fatigue hits). 


12) For BOFU (retargeting)
Answer FAQs & objections in the creative (parking, possession date, approvals, price breakup).
Use testimonials, walkthrough clips, and price-reveal hooks.
Exclude recent leads/bookings; keep BOFU pools distinct (avoid overlap).

0) Prereqs
GTM container installed on all LPs/TP (Init tag first).

1) Event plan (names & parameters)
Core events (required)
form_start — first interaction with the lead form
form_submit_attempt — submit clicked (pre-validation) (optional but useful)
form_submit — validation passed and submit sent
generate_lead — primary conversion (fires once per successful lead)
click_call — tel:+91… link clicked
click_whatsapp — https://api.whatsapp.com/… or https://wa.me/… clicked
brochure_download — brochure file opened/downloading


Shared event parameters (send what you have)


project, city, lp_variant, bhk, price_band, device, page_path, gclid, wbraid, gbraid, landing_url, utm_source, utm_medium, utm_campaign, utm_term, utm_content,utm_matchtype, utm_adgroup, utm_ad, utm_device


Tip: capture UTMs & click IDs in hidden fields on the form so you can pass them reliably.



2) Data Layer (recommended spec)
If you can touch the site code, fire a dataLayer push at the exact moments below. Otherwise you’ll use GTM DOM/Click triggers (next section).


Examples (anywhere after GTM snippet):


<script>
  // On first field focus:
  dataLayer.push({
    event: 'form_start',
    project: 'Vertex 33 West',
    city: 'Hyderabad',
    lp_variant: 'v1',
    bhk: '2/3 BHK',
    price_band: '1.1–1.7 Cr',
    landing_url: location.href
  });

  // On successful submit (before redirect):
  dataLayer.push({
    event: 'form_submit',
    form_id: 'lp-lead-form',
    project: 'Vertex 33 West',
    ...
  });

  // Fire only after the server/API confirms lead created:
  dataLayer.push({
    event: 'generate_lead',
    lead_id: 'abc123',              // if available
    project: 'Vertex 33 West',
    ...
  });
</script>



Why both form_submit and generate_lead?
form_submit = UI success; generate_lead = real lead created (dedup-friendly). In many stacks they’re identical moments; if you can only do one, use generate_lead.




3) GTM build (tags, triggers, variables)



You’ll create one GA4 Event tag per event. All of them use your existing GA4 Config tag.



3.1 form_start
Trigger (choose one):
Preferred (DL event): Trigger type Custom Event → Event name form_start.
No dev option (UI): Trigger type Element Visibility on the form’s first input (CSS selector e.g. #leadForm input[name="name"]), Once per page, or Form Interaction trigger if stable.


Tag: GA4 Event → Event name form_start → Parameters (add variables):
project, city, lp_variant, bhk, price_band, page_path={{Page Path}}, landing_url={{Page URL}}, utm_* , gclid/wbraid/gbraid (from URL vars), utm_source, utm_medium, utm_campaign, utm_term, utm_content,utm_matchtype, utm_adgroup, utm_ad, utm_device




3.2 form_submit_attempt (optional)
Trigger: Form Submission (Some forms) with CSS selector for your submit button or form ID; fire before validation if possible (or on click).
Tag: GA4 Event form_submit_attempt + same parameters.




3.3 generate_lead (mark conversion in GA4)
Trigger:
Preferred: Custom Event generate_lead (push after server/CRM says success).
Thank-You redirect fallback: Page View – Some pages → Page URL contains /tp (or unique slug).
Fallback (no dev): Use the same trigger as form_submit but fire only once (set a 1st-party cookie/sessionStorage flag via a custom script to avoid duplicates on reload).
Tag: GA4 Event generate_lead + parameters.
