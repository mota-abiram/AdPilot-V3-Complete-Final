# Production Setup Guide for AdPilot

## ⚠️ CRITICAL: Credentials Rotation Required

Your `.env` file contains **live API keys that need to be rotated immediately** because they were stored with source code. Follow these steps in order:

---

## Step 1: Rotate All API Credentials

### 1.1 Meta Ads API
- Go to [Facebook Business Manager](https://business.facebook.com/)
- Navigate to Settings → Apps and Websites → System Users
- For your system user, click the token next to your app → Generate New Token
- Copy the new token and save temporarily
- Update `META_ACCESS_TOKEN` in Render dashboard (see Step 3)
- Repeat for `META_REVASA_AURA_ACCESS_TOKEN` if you have a second client

### 1.2 Google Ads API
- Go to [Google Cloud Console](https://console.cloud.google.com/)
- Navigate to APIs & Services → OAuth 2.0 Credentials
- Delete the old OAuth client and create a new one
- Or: Go to Google Ads account → Tools → API Center → Get credentials → Refresh token
- Note down new: `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `GOOGLE_REFRESH_TOKEN`, `GOOGLE_DEVELOPER_TOKEN`
- Update all four in Render dashboard

### 1.3 Anthropic (Claude API)
- Go to [console.anthropic.com](https://console.anthropic.com/) → API Keys
- Deactivate the current key shown in your .env file
- Create a new API key
- Copy it and update `ANTHROPIC_API_KEY` in Render dashboard

### 1.4 OpenAI API
- Go to [platform.openai.com](https://platform.openai.com/) → API Keys
- Revoke the current key shown in your .env file
- Create a new secret key
- Copy it and update `OPENAPI_API_KEY` in Render dashboard

### 1.5 Groq API (if using)
- Go to [console.groq.com](https://console.groq.com/) → API Keys
- Create or refresh your API key
- Update `GROQ_API_KEY` in Render dashboard

---

## Step 2: Update Local .env (Development)

Copy `adpilot/.env.example` to `adpilot/.env`:
```bash
cp adpilot/.env.example adpilot/.env
```

Then fill in the new values you just rotated from Step 1:
```bash
# Use the NEW tokens from Step 1
META_ACCESS_TOKEN=<new_token>
GOOGLE_CLIENT_SECRET=<new_secret>
ANTHROPIC_API_KEY=<new_key>
OPENAPI_API_KEY=<new_key>
etc.
```

**Important:** Do NOT commit `.env` to git. It's already in `.gitignore`.

---

## Step 3: Update Render Dashboard Secrets

Go to [dashboard.render.com](https://dashboard.render.com/) → Your AdPilot Web Service → Environment:

### 3.1 Find the `adpilot-secrets` Environment Group
Click on it to edit.

### 3.2 Update Each Secret with New Values

| Secret | New Value | How to Get |
|--------|-----------|-----------|
| `SESSION_SECRET` | Leave as-is (Render auto-generates) | Already set by Render |
| `DATABASE_URL` | Render Postgres **internal** URL | See Step 3.3 below |
| `META_ACCESS_TOKEN` | New token from Step 1.1 | Meta Business Manager |
| `META_REVASA_AURA_ACCESS_TOKEN` | New token from Step 1.1 | Meta Business Manager |
| `META_AD_ACCOUNT_ID` | (unchanged) | Keep existing |
| `META_REVASA_AURA_AD_ACCOUNT_ID` | (unchanged) | Keep existing |
| `GOOGLE_CLIENT_ID` | New ID from Step 1.2 | Google Cloud Console |
| `GOOGLE_CLIENT_SECRET` | New secret from Step 1.2 | Google Cloud Console |
| `GOOGLE_REFRESH_TOKEN` | New token from Step 1.2 | Google Ads API |
| `GOOGLE_DEVELOPER_TOKEN` | New token from Step 1.2 | Google Ads API |
| `GOOGLE_MCC_ID` | (unchanged) | Keep existing |
| `GOOGLE_CUSTOMER_ID` | (unchanged) | Keep existing |
| `ANTHROPIC_API_KEY` | New key from Step 1.3 | console.anthropic.com |
| `OPENAPI_API_KEY` | New key from Step 1.4 | platform.openai.com |
| `GROQ_API_KEY` | New key from Step 1.5 (if using) | console.groq.com |
| `AUTH_BOOTSTRAP_PASSWORD` | Strong password (e.g. `openssl rand -base64 16`) | Generate new |

### 3.3 Set DATABASE_URL to Render Internal Postgres URL

Go to Render Dashboard → Your PostgreSQL Database → Connection:
- Copy the **Internal Database URL** (looks like `postgres://dpg-xxxxx-a:xxxxx@dpg-xxxxx-a.internal/ad_pilot`)
- Paste it into the `DATABASE_URL` environment variable in the `adpilot-secrets` group
- **Do NOT use the External URL** — the internal URL is faster and doesn't need SSL configuration

---

## Step 4: Generate Strong Session Secret (if needed)

The `SESSION_SECRET` is auto-generated by Render (indicated by `generateValue: true` in render.yaml). If you want to manually set a strong one:

```bash
openssl rand -hex 32
# Output: abc123def456...
```

Then in Render dashboard → Environment → `adpilot-secrets` → `SESSION_SECRET`, paste the generated value.

---

## Step 5: Test the Deployment

1. Go to Render Dashboard → Your Web Service → Manual Deploy → Deploy latest commit
2. Wait for deployment to complete (logs show "Your service is live")
3. Visit `https://your-adpilot.render.com/` 
4. Try logging in with:
   - Email: The value of `AUTH_BOOTSTRAP_EMAIL` (default: `admin@adpilot.local`)
   - Password: The value of `AUTH_BOOTSTRAP_PASSWORD` (the new strong one from Step 3)
5. On dashboard, click "Run Agent" and verify the sync works

---

## Step 6: Verify Database Connection

The database connection should automatically use the internal Render Postgres URL (no SSL issues). Verify in the logs:

```bash
# In Render logs, you should see:
[DB] Connected to Postgres via pool
[DB] Unexpected pool error: (should be none)
```

If you see SSL errors, ensure:
- `DATABASE_URL` is the **internal URL** (starts with `postgres://dpg-`)
- Not the external URL (contains `.render.com`)

---

## Step 7: Disable the Old Bootstrap Admin (Optional but Recommended)

Once you've logged in with the new admin password, you should prevent other users from creating accounts without authorization:

1. In the app, go to Settings → Users (if admin panel exists)
2. Note the auto-created admin user
3. The next-user-to-signup will need an invite (if role-based access is enforced)

---

## Security Checklist

- [ ] All API keys from Step 1 have been rotated
- [ ] `.env` file is in `.gitignore` and not committed to git
- [ ] `DATABASE_URL` in Render is set to the **internal** Postgres URL
- [ ] `SESSION_SECRET` is a strong random value (not `adpilot-local-session-secret`)
- [ ] `AUTH_BOOTSTRAP_PASSWORD` is changed from `change-me-123` to something strong
- [ ] All 6 environment variables (`ANTHROPIC_API_KEY`, `OPENAPI_API_KEY`, `GROQ_API_KEY`, and Meta/Google tokens) are in the Render `adpilot-secrets` group
- [ ] Web service is on `starter` plan or higher (not `free`)
- [ ] PostgreSQL database is on `Starter` plan or higher (not free tier with 90-day TTL)
- [ ] Deployment succeeds without errors
- [ ] You can login and dashboard loads data

---

## Ongoing Security Maintenance

1. **Rotate API keys every 90 days** — set a calendar reminder
2. **Review Render logs weekly** — watch for connection errors, auth failures
3. **Monitor your bill** — if using free tier Postgres, upgrade before 90 days
4. **Enable 2FA** on Meta, Google, Anthropic, and OpenAI accounts
5. **Never commit credentials** — if you accidentally commit a key, rotate it immediately

## Troubleshooting

### "DATABASE_URL must be set in production"
- Go to Render Dashboard → your web service → Environment → `adpilot-secrets` 
- Ensure `DATABASE_URL` is set to your internal Postgres URL
- Redeploy

### "SSL: CERTIFICATE_VERIFY_FAILED"
- This means you're using an external URL instead of internal
- Go to Render Dashboard → your Postgres database → Connection → copy the internal URL
- Update `DATABASE_URL` in Render to use the internal URL
- Redeploy

### "Login fails / session lost after refresh"
- Ensure `SESSION_SECRET` is set in Render and is a strong random value
- Ensure `DATABASE_URL` is valid and the DB is reachable
- Check server logs for "session" errors

### "Agent sync fails / no data in dashboard"
- Ensure all Meta and Google credentials are current (rotated from Step 1)
- Ensure `ANTHROPIC_API_KEY` and `OPENAPI_API_KEY` are set in Render
- Check Render logs for "401 Unauthorized" or "Invalid token" errors
- If you see credential errors, re-run Step 1 and update Render secrets
